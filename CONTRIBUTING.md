# Contributing

## Development Workflow

- Work on feature branches for larger changes: `feature/<name>`
- Small commits may go directly to main
- Test changes before pushing
- Update documentation for any changed behavior

## Definition of Done

A task is complete when:

- Code runs without errors and functions as intended
- Documentation is updated if behavior changes
- Tests pass

## Pull Requests

- Large features require one approving review
- Reviewers verify the change meets the Definition of Done
- All reviews tracked through GitHub pull requests
